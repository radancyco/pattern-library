Accessibility & UX

Jump Menus that utilize a select dropdown to link to external pages, must always include a submit button. Unexpectedly taking a user to another page, upon making a selection from an input control, is a "change of context", and therefore a failure of Success Criterion 3.2.2: On Input. The only exception to this rule is in-page navigation. If you are using a select dropdown to change content wihin the page, then you do not need a submit button. However, you do need to announce that a change has occured via aria-live.

If you want to navigate away from a page without a submit button, then simply use a hyperlink or a disclosure that reveals a list of hyperlinks to other pages, when opened.

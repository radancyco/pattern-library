Advisory

Please be advised that the current demo reflects implementation of the picture element before Internet Explorer 11 support was dropped. This behavior is also currently reflected in the product and will be updated at a later date to support a "mobile first" approach.

Ideally, the default image should be the smallest, with single or multiple source elements containing progressively larger images. Each source would contain a media attribute with min-width value set for each breakpoint to reval image at.

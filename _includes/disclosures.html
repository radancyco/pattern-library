Accessibility & UX

Toggling or changing text in a control to indicate a state change or another action to be taken should be avoided. While it is not out of realm of possibility to make such a control accessible, doing so takes additional time and may introduce unnecessary complexity. For example, don't do this:

  • Changing the text removes the initial context of the control, which can be confusing, especially if aria-expanded or aria-pressed is used, for assistive technology users.
  • Instead of changing text, consider using visual affordance to convey that toggling behavior exists, which is a well-known and quickly understood convention. For example, up and down arrows, check mark, plus and minus, or close ("X") indicators.
  • Using a single instruction means their is less text to translate.
  • Users are well-adept at using such interfaces. Afterall, if they knew how to open it, a text instruction to close it is really unnecessary.

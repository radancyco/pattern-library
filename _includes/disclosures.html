Accessibility & UX

Toggling text in a control to indicate a state change should be avoided. For example, don't do this:

  • Changing text removes the initial context of the control, which can be confusing, especially if aria-expanded or aria-pressed are used to aid assistive technology users.
  • Changing text may not announce the state change to assistive technology users.
  • Instead of changing text, use visual affordance to convey that toggling behavior exists. This is a well-known and quickly understood convention. For example, up and down arrows, check mark, plus and minus, etc.
  • Using a single instruction means their is less text to translate.
  • Users are well-adept at using such interfaces. Afterall, if they knew how to open it, a text instruction to close it is really unnecessary.

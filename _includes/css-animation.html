Responsible Animation

Be mindful of the impact that animation can have on people with disabilities. Animation can be a distraction for some, make others ill, and can even be dangerous for those prone to seizures. By adhering to the following rules, we can make our experiences pleasant and safe for everybody!

  • If animation loops continuously, such as a hero banner video, background animation, auto-scrolling carousel, etc. then it must include a pause button. A component is available to easily implement a pause button on CSS and video animations. Third-party plugins, such as Slick, typically have their own implementation of pause.
  • Micro-animations, such as those which occur when scrolling or interacting with page elements, are exempt if they do not last for more than 5 seconds.
  • Developers should honor user settings by using the prefers-reduced-motion media query to reduce or eliminate animation. You can learn more about this media query on MDN.
  • Animations that flash for more than 3 times per second are prohibited on our websites, as they may induce seizures. A pause button will be ineffective here. If you suspect an animation of having a high flash rate, please bring this to the attention of the Accessibility Team.

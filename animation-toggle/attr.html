Properties

Use the following properties to configure this component.

Classes
Class Note Description
.animation-toggle Mandatory Include this class on the parent element that wraps your content and animation.
.animation-toggle__button Mandatory This class is included on all pause buttons generated by the script.
.animation-toggle__video Mandatory Include this class on all video elements if used.
.animation-enabled Mandatory This class is applied to the body element on page load. CSS animation will only run when this class is present. Removing it will pause animation. By default, we are putting accessibility first by treating animation as an enhancement.
.lazy-load Optional This class is applied to the video element if lazy loading is desired.
Attributes
Attribute Note Description
crossorigin Boolean, Optional Include this attribute on any video element in use. Indicates whether to use CORS to fetch the related video, which is often needed for cross-domain hosting of assets.
data-src Optional Include this attribute on any video source element that utilizes lazy loading. It's value should be the path to your video asset.
loop Boolean, Mandatory Include this attribute on any video element in use. Used to continulously loop the video.
muted Boolean, Mandatory Include this attribute on any video element in use. Permanetly mutes video. Without it, video will not autoplay.
playsinline Boolean, Mandatory Include this attribute on any video element in use. Indicates that the video is to be played inline, within the element's playback area
Elements
Element Note Description
video Optional Include this element if choosing to include animation via HTML video. See allowed attributes for more details.

Notes

  • Video should always be delivered as an optimized webm or mp4. The file size should be as minimal as possible without sacrificing quality, to reduce overall page weight.
  • Video should be no longer than 15-30 seconds in length. The shorter the video is, the less of an impact it will have on performance.
  • In addition to the pause button, video and CSS animations utilize prefers-reduced-motion. When a user disables animation in their operating system, the animation will not autoplay, nor will the pause button be present.
  • A cookie called AnimationPaused is added and removed when pause button is toggled. This cookie remembers the desired setting on other pages of the site that use the same script and on return visits.
  • Video may not autoplay if mobile device is in low-power mode. This is to preserve battery life.

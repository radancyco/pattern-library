Properties

Use the following properties to configure this component.

Attributes
Attribute Apply Note Description
aria-label video String, Optional This will override the default label, which is "Background Animation".
crossorigin video Boolean, Optional Indicates whether to use CORS to fetch the related video, which is often needed for cross-domain hosting of assets.
data-large-viewport video String, Optional Include when breakpoint is needed to load in larger video asset. It's value should be the path to your large screen video asset.
data-media video String, Optional Include when breakpoint is needed to load in larger video asset. It's value can be any valid media query.
data-pause-button .animation-toggle String, Optional This will override the default button label, which is "Pause Animation".
data-src source Optional Include when lazy loading required. It's value should be the path to your video asset.
loop video Boolean, Mandatory Used to continulously loop the video.
muted video Boolean, Mandatory Permanetly mutes video. Without it, video will not autoplay.
playsinline video Boolean, Mandatory Indicates that the video is to be played inline, within the element's playback area
Classes
Class Apply Note Description
.animation-toggle div Mandatory Include on the parent element that wraps your content and animation.
.animation-toggle__button button Mandatory This class is included on all pause buttons generated by the script.
.animation-toggle__video video Mandatory Include this class on all video elements if used.
.animation-enabled body Mandatory This class is applied on page load. CSS animation will only run when this class is present. Removing it will pause animation. By default, we are putting accessibility first by treating animation as an enhancement.
.lazy-load video Optional Include when lazy loading is required.
Elements
Element Note Description
video Optional Include this element if choosing to include animation via HTML video. See allowed attributes for more details.

Notes

  • Video should always be delivered as an optimized webm or mp4. The file size should be as minimal as possible without sacrificing quality, to reduce overall page weight.
  • Video should be no longer than 15-30 seconds in length. The shorter the video is, the less of an impact it will have on performance.
  • In addition to the pause button, video and CSS animations utilize prefers-reduced-motion. When a user disables animation in their operating system, the animation will not autoplay, nor will the pause button be present.
  • A cookie called AnimationPaused is added and removed when pause button is toggled. This cookie remembers the desired setting on other pages of the site that use the same script and on return visits.
  • Video may not autoplay if mobile device is in low-power mode. This is to preserve battery life.
  • Be mindful on translation of video and pause button on multi-language sites.

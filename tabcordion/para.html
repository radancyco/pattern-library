Parameters

  • data-tab-active (Number, Optional): Using a number from "1" up, you can specify which tab and disclosure (accordion) should be active on page load. When this attribute is not specified, the first panel will be open by default. A tab or disclosure must always have at least one panel open. Having all disclosures collapsed in smaller viewports is not currently supported.
  • data-tab-animation (Boolean, Optional) : This attribute will enable animation of the content on page load and when tab or disclosure button is selected. You may customize the default CSS animation or keep what is currently in place.
  • data-tab-breakpoint (String, Required): Accepts any valid media query for more granular control over when interface needs to change from disclosure to tab and vice versa. Setting the breakpoint in code is important and allows us to pass in the needed roles and ARIA attributes for each interface type.
  • data-tab-disable-url (Boolean, Optional): By default, the URL will always change to reflect selected page fragment. This is recommended behavior because it allows us to share fragmented content more easily. If this behavior is not desired, you may use this attribute to disable this functionality.
  • data-tab-vertical (Boolean, Optional) : This attribute will change the tab interface from horizontal, the default, to vertical. Placement of tabs on the right side of the the content is not currently supported. If buttons are moved to this position, keyboard arrow functionality will be reversed.

This version of the feed may be used to generate a list of videos or playlists from an individual YouTube account. The feed can be linked to directly or referenced within an inline frame (iframe).

This feed leverages version 3 of the YouTube Data API.

Note: A modal version of this feed will be released at a later date. Should your project require a modal, a custom solution will be needed.

Initial Setup

While the feed can be viewed directly, the most common use case is to load this content within an iframe element, by supplying an absolute path to the feed application. A title attribute and value must also be provided.

<iframe loading="lazy" src="https://services.tmpwebeng.com/tmp-share/widget/youtube-feed-iframe.aspx" title="Videos from [Company Name]"></iframe>

However, the feed will not render usable content until we begin passing parameters to it.

Accessibility

Please note use of the title attribute in all code examples. A brief description of the contents of the iframe element must be provided to assist those who are using assistive technology, such as screen readers, to access the content.

Parameters & Examples

The following is a list of parameters that can be passed to the feed to render content and to allow for better control over its presentation and behavior. Code and output examples provided where applicable.

autoplay

Boolean: When set to true, the video will automatically begin playing on those browsers that support it. Many mobile devices do not allow video to automatically play to conserve a user's bandwidth. The default is false.

<iframe loading="lazy" src="https://services.tmpwebeng.com/tmp-share/widget/youtube-feed-iframe.aspx?ChannelName=Radancy&autoplay=true" title="Videos from {{ site.company-name }}"><lt;/iframe>

Accessibility

Setting the language of a document is very important in aiding those who use assistive technology, such as screen readers. Defining the proper language attribute and value (<html lang="en">, for example) satisfies WCAG 2.0 Level A guidelines. Please see 3.1.1 Language of a Page for more details.

tab

Boolean: When set to false, the video or direct link to YouTube.com, will play in same tab or window. The default is true.

Note: A direct link to YouTube.com will not open within an iframe.

<iframe loading="lazy" src="https://services.tmpwebeng.com/tmp-share/widget/youtube-feed-iframe.aspx?ChannelName=Radancy&tab=false" title="Videos from {{ site.company-name }}"></iframe>

View in new window

target

Boolean: When set to true, the video link will open in a new tab or window on YouTube.com rather than the default behavior, which is for the video to consume an entire tab or window. The default is false.

<iframe loading="lazy" src="https://services.tmpwebeng.com/tmp-share/widget/youtube-feed-iframe.aspx?PlayListID=PLS8wCS0z7X-Z2sUR3IOMFWM8uodogBiST&target=true" title="RWD School of Rock Sessions"></iframe>

Output

Depending on the parameter that is passed in via the query string, the HTML output rendered within the iframe element will be formatted in one of two ways:

Figure & Captions

This is the default markup when no parameters, outside of ChannelName or PlayListID, are passed in.

<div id="youtube-feed">

<a href="https://www.youtube.com/embed/CEcfakeuAbg">

	<figure>

		<img src="https://i.ytimg.com/vi/CEcfakeuAbg/mqdefault.jpg" alt="">

		<figcaption>Internet vs. Cupcake III <span class="wai">(Link opens in new window)</span></figcaption>

	</figure>

</a>

...

</div><!--/#youtube-feed-->

Embed

When the embed parameter is passed in, the output will resemble the code listed below.

<div id="youtube-feed">

	<figure>

		<div>

			<iframe loading="lazy" src="https://www.youtube.com/embed/hs0O2gGMkTg?autohide=1&showinfo=0" allowfullscreen></iframe>

		</div>

		<figcaption>RWD School of Rock: Fall 2013 - Session 1a (1)</figcaption>

	</figure>

</a>

...

</div><!--/#youtube-feed-->

List

When images are disabled, the markup will be output as a list. See the image parameter for more details.

<div id="youtube-feed">

	<ul>

		<li><a href="https://www.youtube.com/embed/CEcfakeuAbg">Internet vs. Cupcake III <span class="wai">(Link opens in new window)</span></a></li>

		...

	</ul>

</div><!--/#youtube-feed-->

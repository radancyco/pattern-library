This pattern will allow you to easily implement and control the presentation of a button grid with multiple disclosures.

Note: Do not use this if you are looking for a simple, tabbed interface. Instead, use the TabCordion pattern.

Parameters

  • data-grid-active (Number, Optional): Using a number from "1" up, you can specify which content area should be open on page load. When this attribute is not specified, no panels will be open.
  • data-grid-callback (String, Optional): You can use a callback to initiate a function within the disclosed content area on page load and reinitiate the same function on button click. The function will fire when any button in the grid disclosure is engaged with. The value of this data attribute should, of course, be the same as the function you wish to call.
  • data-grid-close (Boolean, Optional): Removes all close buttons from content area.
  • data-grid-disable-url (Boolean, Optional): By default, the URL will always change to reflect selected content area. This is recommended behavior because it allows one to share and find fragmented content more easily. If this behavior is not desired, you may use this attribute to disable this behavior.
  • data-grid-large (String, Optional): Your "large" viewport breakpoint. At this breakpoint, your layout will be thrown into "grid mode".
  • data-grid-max (Number, Mandatory): Specifies the number of grid items to display per row in "large" viewport. This attribute is mandatory when data-grid-large is in use.
  • data-grid-medium (String, Optional): Your "medium" viewport breakpoint. At this breakpoint, your layout will be thrown into "grid mode".
  • data-grid-min (Number, Mandatory): Specifies the number of grid items to display per row in "medium" viewport. This attribute is mandatory when data-grid-medium is in use.
  • data-grid-sticky (String, Optional): Accepts a single parameter; the ID value of the fixed position element, such as your header, that you may wish to offset the grid layout from. New!
  • data-grid-exclude (Boolean, Optional): To be placed on both the button and content you wish to not be included in the grid. This is particulary useful in situations where you may wish to dynamically hide content. Please note that if you are dynamically adding or removing a grid item, that the gridViewPort() function will need to be reinitiated.

Coming Soon

  • Ability to include placeholder content to fill in empty grid spaces. Some clients (and designers) do not like uneven space in desktop design.
  • Allow mobile accordions to have multiple disclosures open instead of one at a time.
  • Allow mobile accordions to all be closed.
  • Ability to customize URL fragments. For example, #about-us-1-1 instead of the generic #grid-content-1-1.

This pattern will allow you to easily place and control the presentation of a grid with multiple disclosures.

Parameters

  • data-grid-active (Number, Optional): Using a number from "1" up, you can specify which content area should be open on page load. When this attribute is not specified, no panels will be open.
  • data-grid-close (Boolean, Optional): Removes all close buttons from content area.
  • data-grid-breakpoint (String, Required): Accepts any valid media query for more granular control over when interface needs to change from accordion to tab. Setting the breakpoint in code also allows us to pass in the required roles and ARIA attributes needed for each type of interface. For example, setting (min-width: 800px) and (max-width: 1200px) would only allow tabs between these two selected breakpoints. See example below.
  • data-grid-callback (String, Optional): You can use a callback to initiate a function within the disclosed content area on page load, or reinitiate the same function on button click. The function will fire when any button in the grid disclosure is engaged with.
  • data-grid-disable-url (Boolean, Optional): By default, the URL will always change to reflect selected content area. This is recommended behavior because it allows one to share fragmented content more easily. If this behavior is not desired, you may use this attribute to disable this functionality.

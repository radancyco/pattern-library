Use Case #1 - Create Banners to use with Content Page Settings

The default code used on site themes includes the structure that you need to leverage those page-level items.

Use Case #2 - Create Banners with Section & Modules

For core pages where you need to create a section and code to support a hero. Create a section with (2) partials.

Partial 1: Should contain your images/media modules wrapped in a picture element.

  1. Mobile image should be first (Source)
  2. Desktop image should be second (IMG)

Partial 2: Should contain your Text/HTML module. Make sure it's setup for SEO to use the right headline element (h1, h2, etc.)

SCSS Variables

  • $end-banner This is your breakpoint for when you want to go from mobile to desktop.
  • $banner-constrain-wrapper If you need to contain the headline that overlays your hero image with a max-width to match other margins on the page. Used with .constrain class. (example: 1400px)
  • $banner-x-offset Will allow you to choose the horizontal distance you want for your text to be from left or right. This would not apply to horizontally centered classes below.
  • $banner-y-offset Will allow you to choose the vertical distance you want for your text to be from top or bottom. This would not apply to vertically centered classes below.

CSS Classes

The following classes can be used to help determine the orientation and position of the text

  • .constrain This will apply a max-width around your text's inner-wrapper so that it can stay within margins used on page. Default would have text area be 100% the width of the viewport.
  • .text-left-top This will align text with top (vertical) left(horizontal) of the hero.
  • .text-center-top This will align text with top center of the hero.
  • .text-right-top This will align text with top right of the hero.
  • .text-left-center This will align text with center left of the hero.
  • .text-center-center This will align text with center (horizontal and vertical) of the hero.
  • .text-right-center This will align text with center right of the hero.
  • .text-left-bottom This will align text with bottom left of the hero.
  • .text-center-bottom This will align text with bottom center of the hero.
  • .text-right-bottom This will align text with bottom right of the hero.

Accessibility

Before using an h1 heading in your hero banner, please make absolutely sure it is the right heading to use.

  • Is there another element on the page that is better suited to be a primary heading? Often, this will be below the hero banner. If so, this should be your h1.
  • Is the hero banner meant as more of a call to action or branded message? If so, you may wish to treat the banner as decorative, in which case a div could be used instead of an h1 or lower heading level.
  • Think contextually. Often, the hero banner on the homepage could justifiably describe the overarching message of the site and be an h1, while hero banners on subpages can contain a div, especially if those suppages include additional and more suitable headings that can be the h1.

If you are not sure, then please reach out to Michael Spellacy.

Advisory

Please be advised that the current demo reflects implementation of the picture element before Internet Explorer 11 was dropped. This behavior is also currently reflected in the product and will be updated at a later date to support a mobile first approach.

Ideally, the default image should be your smaller image and the media attribute should reflect a min-width value.

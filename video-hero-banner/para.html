Parameters

  • hero-banner (Mandatory): The primary id and hook for CSS and JavaScript.
  • data-banner-breakpoint (String, Mandatory): Accepts any valid media query. This attribute is mandatory when data-banner-desktop and data-banner-mobile are in use.
  • data-banner-mobile (String, Optional): Accepts a valid URL to your mobile video. This video will load on smaller viewports, as specified by the breakpoint value used in data-banner-media.
  • data-banner-desktop (String, Optional): Accepts a valid URL to your desktop video. This video will load on larger viewports, as specified by the breakpoint value used in data-banner-media.
  • data-banner-all (String, Optional): Accepts a valid URL to the video you wish to use across all viewports.
  • data-banner-caption (String, Optional): Accepts three, comma-delimited values to include captions. These are the path to the VTT file, the ISO two letter language code, and a label to identify the caption. Please note that loading captions are the only way to unmute the video. This is by design to enforce accessibility. Video with captions will also enable default video controls and remove the single pause button.
  • data-banner-description (String, Optional): Accepts three, comma-delimited values to include audio descriptions. These are the path to the VTT file, the ISO two letter language code, and a label to identify the descriptions. Please note that audio descriptions are not currently supported in browsers, but you can test them out using a browser plugin.

Notes

Video may not load or appear paused when a device is in "Low Power Mode". This is not a bug but a feature of the device to save power, which video can drain a lot of.
